发表于 2017-08-19 | 阅读次数:

首先我是在这里看到的教程，紧跟着去了另一个站,然后想自己做一个，走过了一路的坑，关于各种软件的介绍我不在赘述，下面进入正文。

准备过程

首先你电脑需要安装git，然后安装软件node直接去官网下载安装，我安装的版本已经自带了npm所以不用自己安装。上面两个软件都安装好之后就是安装hexo了

运行命令尽量使用 Git bash 尽量不要使用自带的cmd，否则有可能出现意想不到的错误。无法写入或者安装

然后查看是否安装完成，这时候为第一个坑埋下了伏笔、、、、

查看安装完成之后就可以创建一个新的hexo项目了，

然后你会发现在当前目录下多一个你项目名的文件夹，然后就可以启动本地服务查看一下效果了# 进入目录

有关hexo目录的介绍以及配置文件的介绍，请查看我引用的第二个网址,然后就是各种页面修改，以及主题美化了，这些都是一些界面的配置，具体请百度，这里不赘述。

下面坑开始了。睁大眼睛哦

至此第一个坑爬过了。

Second

然后就是将域名附加到我们的github仓库了，首先申请一个域名，然后配置域名解析 我申请的是阿里云的域名，然后这是域名解析配置

下面就是配置github的CNAME了，首先在主分支中新建一个CNAME文件没有后缀，我的项目当时有两个个分支一个master一个gh-pages，gh-pages分支是因为我看教程上面的是这个，所以我才加了一个，然后我添加CNAME到master而主分支是gh-pages，然后就是各种404访问不到，如果你分支比较多记得在主分支中新建CNAME，然后在里面写上你的域名就一行

这里就是第二个坑了 ，一定在主分支加CNAME
其实还可以在设置里面设置，Setting->Options找到GitHub Pages 然后设置域名就可以了
